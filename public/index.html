Receptenweb API

RESTful API Documentatie - Node.js & Express

Base URL: http://localhost:3000/api
Versie: 1.0.0
Response Format: JSON

News Endpoints

GET /api/news

Haal alle news items op met paginatie

Query Parameters:

  • limit (optioneel) - Aantal items per pagina (default: 10)
  • offset (optioneel) - Aantal items om over te slaan (default: 0)

Voorbeeld Request:

GET /api/news?limit=5&offset=0

Voorbeeld Response:

{
  "success": true,
  "data": [
    {
      "id": 1,
      "title": "Nieuwe recepten toegevoegd",
      "content": "We hebben 10 nieuwe recepten...",
      "author": "Admin",
      "created_at": "2024-01-07 12:00:00",
      "updated_at": "2024-01-07 12:00:00"
    }
  ],
  "pagination": {
    "total": 25,
    "limit": 5,
    "offset": 0,
    "hasMore": true
  }
}

GET /api/news/:id

Haal één specifiek news item op

Voorbeeld Request:

GET /api/news/1

GET /api/news/search

Zoek news items op basis van titel of content

Query Parameters:

  • q (verplicht) - Zoekterm
  • limit (optioneel) - Aantal resultaten (default: 10)
  • offset (optioneel) - Offset voor paginatie (default: 0)

Voorbeeld Request:

GET /api/news/search?q=recept&limit=5

POST /api/news

Maak een nieuw news item aan

Request Body:

{
  "title": "Titel van het nieuws",
  "content": "Inhoud van het nieuws",
  "author": "Auteur naam" (optioneel)
}

PUT /api/news/:id

Update een bestaand news item

Request Body:

{
  "title": "Nieuwe titel",
  "content": "Nieuwe inhoud",
  "author": "Nieuwe auteur"
}

DELETE /api/news/:id

Verwijder een news item

Voorbeeld Request:

DELETE /api/news/1

Posts Endpoints

GET /api/posts

Haal alle posts op met paginatie

Query Parameters:

  • limit (optioneel) - Aantal items per pagina (default: 10)
  • offset (optioneel) - Aantal items om over te slaan (default: 0)

GET /api/posts/:id

Haal één specifieke post op

GET /api/posts/search

Zoek posts op basis van content

Query Parameters:

  • q (verplicht) - Zoekterm
  • limit (optioneel) - Aantal resultaten (default: 10)
  • offset (optioneel) - Offset voor paginatie (default: 0)

POST /api/posts

Maak een nieuwe post aan

Request Body:

{
  "content": "Inhoud van de post",
  "user_id": 1 (optioneel)
}

PUT /api/posts/:id

Update een bestaande post

Request Body:

{
  "content": "Nieuwe inhoud"
}

DELETE /api/posts/:id

Verwijder een post

Error Responses

Error Format

Alle errors worden geretourneerd in het volgende formaat:

{
  "success": false,
  "error": "Foutmelding",
  "message": "Gedetailleerde foutbeschrijving"
}

HTTP Status Codes:

  • 200 - OK
  • 201 - Created
  • 400 - Bad Request (validatie fout)
  • 404 - Not Found
  • 500 - Internal Server Error
